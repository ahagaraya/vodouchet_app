## Следующие итерации (production)

| Приоритет | Задача | Зачем |
|-----------|--------|-------|
| 🔴 Высокий | Облачный backend + HTTPS | Не зависеть от IP ПК |
| 🔴 Высокий | Настройка URL API в приложении | Без пересборки APK |
| 🔴 Высокий | PostgreSQL вместо SQLite | Масштабирование, конкурентные записи на production |
| 🟡 Средний | Push-уведомления | Новые заявки, сообщения чата |
| 🟡 Средний | Офлайн-кэш каталога | Работа при слабом интернете |
| 🟢 Низкий | CI/CD (GitHub Actions) | Автосборка и тесты |
| 🟢 Низкий | Интеграция 1С / CRM | Учёт компании |
| 🟢 Низкий | Публикация в Google Play (AAB) | Профиль `production` в `eas.json` |
| 🟢 Низкий | Биометрия на входе | Безопасность на устройстве |

## Технический долг (известные ограничения)

- Backend только на локальном ПК — для демо на защите.
- `CI=1` при web-запуске через Expo — без hot reload, нужен перезапуск.
- Web-карта — ссылка на Google Maps, не native SDK.
- SQLite одним файлом — надёжнее JSON, но не замена PostgreSQL при высокой нагрузке записи.
- `node:sqlite` — экспериментальный API Node.js (возможно предупреждение в консоли).

Подробнее в [defense-qa.md](defense-qa.md), раздел «Ограничения».
