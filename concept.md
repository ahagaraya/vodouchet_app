# Концепция: приложение для выездных специалистов ООО «Водоучет»

Описание ролей, бизнес-процесса и ключевых сущностей системы.

См. также: [README.md](README.md), [structure.md](structure.md), [defense-qa.md](defense-qa.md).

---

## Назначение

Единое мобильное приложение для:

- **клиентов** — оформление заявок на выезд, отслеживание заказов, отзывы, чат с администратором;
- **выездных специалистов** — список назначенных выездов и отчёт на объекте;
- **администратора** — согласование заявок, формирование заказов, отчётность, поддержка.

---

## Роли

### 1. Клиент (`client`)

**Раздел:** «Мои заявки» (вкладка в нижней навигации).

**Возможности:**

- регистрация с подтверждением email (только роль client);
- создание заявки: адрес, описание, дата выезда, позиции из каталога, желаемый специалист;
- автозаполнение адреса и телефона из профиля;
- редактирование и **отмена заявки** (с подтверждением «Да / Нет») до перевода в заказ;
- ответ на предложение цены от администратора;
- запрос переноса даты выезда;
- просмотр заказа: таймлайн статусов, чат по заказу, согласование доп. работ;
- отзывы по завершённым заказам, претензии;
- история обращений;
- онлайн-чат с администратором (виджет на всех экранах);
- **сортировка** списка: новые, старые, по активности, по дате выезда.

**Демо:** `anna_k` / `Admin1234`

---

### 2. Выездной специалист (`employee`)

**Раздел:** «Выезды».

**Видит только** заказы со статусами `published`, `in_progress`, `completed`, **назначенные ему** (`assigned_user_id`).

**Карточка выезда:**

- адрес (отдельный блок);
- дата и время, стоимость;
- контакт на объекте (имя, телефон);
- перечень работ (чеклист);
- комментарий клиента;
- статус (`StatusBadge`).

**После выполнения:**

- отметка работ выполнено / не выполнено;
- статус заказа: `in_progress` → `completed`;
- статус оплаты, служебная отметка;
- при невозможности выполнить — причина отказа (`no_access`, `defect`, `client_absent`, `other`);
- добавление **дополнительных работ** (с согласованием клиента).

**Сортировка** списка выездов — те же режимы, что у клиента и админа.

**Демо:** `igor_m`, `sergey_p` / `Admin1234`

---

### 3. Администратор (`admin`)

**Раздел:** «Админ» — горизонтальные вкладки:

| Вкладка | Назначение |
|---------|------------|
| Сводка | Дашборд: новые заявки, просрочки, выручка |
| Отчёты | Выезды и выручка за месяц |
| Заявки | Входящие (сайт, Telegram, приложение) |
| Заказы | Все выезды, редактирование |
| Каталог | CRUD категорий и позиций |
| Журнал | Аудит действий |
| Претензии | Жалобы клиентов |
| Поиск | По заказам и клиентам |
| Клиенты | Карточки клиентов |
| Создать / Редактировать | Форма заказа + чат по заказу |
| Чаты | Переписка (бейдж непрочитанных) |

**Демо:** `admin` / `Admin1234`

## Бизнес-процесс

```
Клиент (сайт / Telegram / приложение)
        ↓
   incomingRequests  (status: new → negotiating)
        ↓
   Администратор: согласование, предложение цены, перенос даты
        ↓
   Создание заказа + назначение специалиста
        ↓
   publish → orders.status: published
        ↓
   Мобильное приложение специалиста («Выезды»)
        ↓
   Отчёт на объекте: in_progress → completed
        ↓
   Клиент: отзыв, документы в истории
```

**Важно:** заявка **не попадает** к мастеру автоматически — только после действия администратора (создание и публикация заказа).

---

## Статусы

### Заявка (`incomingRequests.status`)

| Статус | Значение |
|--------|----------|
| `new` | Новая |
| `negotiating` | Согласование |
| `confirmed` | Переведена в заказ |
| `rejected` | Отклонена |

### Заказ (`orders.status`)

| Статус | Значение |
|--------|----------|
| `draft` | Черновик (не виден специалисту) |
| `published` | У специалиста |
| `in_progress` | В работе |
| `completed` | Выполнен |
| `cancelled` | Отменён |

## Обратная связь

- **Общий чат** — клиент ↔ администратор (`SupportChatWidget`).
- **Чат по заказу** — участники заказа (`OrderChatBlock`, `ChatPanel`).
- **Отзывы** — клиент после `completed`.
- **Претензии** — эскалация проблем (`complaints`).

## API (основное)

| Метод | Путь | Кто |
|-------|------|-----|
| GET | `/api/orders` | специалист — свои выезды |
| GET/PATCH | `/api/orders/:id/report` | отчёт специалиста |
| GET/PATCH | `/api/client/requests` | клиент — заявки |
| GET | `/api/admin/requests` | админ — заявки |
| GET/POST/PATCH | `/api/admin/orders` | админ — заказы |
| POST | `/api/public/requests` | заявка с сайта |
